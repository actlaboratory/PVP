# 初めに

Parametric Video Processor は、スクリーンリーダーを利用している環境で、簡単な動画編集を可能にします。ちまたの動画編集ソフトは、読み上げに対応していないものが多く、動画の不要部分をカットすることさえ難しいです。そんな現状をちょっとでもなんとかするために作りました。

# 基本的な操作方法
操作はとても簡単です。
- PVPを起動します。
- やりたいことを選びます。
- 必要な情報をダイアログ画面でポチポチ設定します。
- 完了ボタンを押します。
- あとはいい感じにやってくれます。

# できること
現在は、以下のことができます。これから、自分が欲しくなったり、要望をいただいたりした際には、機能が増えるかもしれません。

## 動画のカット編集
動画の不要部分をカットすることができます。最初と最後を切るとか、間で打ち合わせをしちゃっていらなくなったところを切るとか、いわゆる普通のカット編集を行えます。
最初に、カット編集したい動画ファイルを選びます。
次に、編集画面で、カットしたいポイントを設定します。編集画面の操作は、次の章で説明します。
最後に、出力先を選びます。

## 音声ファイルをTwitter投稿できる形式に変換
おまけ機能です。Twitterに動画を上げるには、mp4ファイルに変換する必要があります。しかも、なんか内部形式の細かい指定があって、これを満たしていないと怒られてしまいます。そういうのをいい感じにやってくれます。
最初に、変換したい音声ファイルを選びます。
次に、画面に表示する画像ファイルを選びます。これは、動画にする都合上しょうがないので、適当な画像サイトとか、自分の家の猫の写真とかを用意してください。
最後に、出力先を選びます。
PVP は、指定された音声ファイルを最初から最後まで変換しますが、音声が長すぎると、 Twitter に上げられない可能性があります。 Twitter が動画の長さを制限しているからです。 Twitter はいろんな制限をころころサイレントに変えることで有名なので、 PVP のほうで動画の長さを制限したり切り取ったりするような気遣いをしていません。
※ Twitter は、 X と呼ばれることもあります。

# カット編集画面の使い方
基本的には、ボタンを使って操作していきます。
「再生」ボタンを押すと、動画が再生されます。「再生」ボタンは、「停止」ボタンに変わります。
「停止」ボタンを押すと、現在の位置で再生が止まります。「停止」ボタンは、「再生」ボタンに変わります。
「進める」と「戻す」ボタンがあります。これを押すと、再生位置を進めたり戻したりできます。この間隔を、次の「間隔調整」ボタンで変えることができます。
「間隔調整」ボタンを押すと、メニューが出てきます。ここでは、「進める」と「戻す」ボタンを押したときの移動間隔を調節できます。
「指定時間へ」ボタンを押すと、指定したところに再生位置を移動できます。「5」と入力すると5分のところに、「5:30」と入力すると5分30秒のところに、「1:20:30」と入力すると1時間20分30秒のところに、「1:20:30.400」と入力すると1時間20分30.4秒のところに移動します。
「ここからカット」ボタンを押すと、カット位置が設定されます。これを押したときに再生されていた位置から動画の最後までがカット対象になります。「ここからカット」は、「ここまでカット」ボタンに変わります。
「ここまでカット」ボタンを押すと、さっき「ここからカット」を推した位置から現在再生している位置までがカット対象になります。「ここまでカット」は、「ここからカット」ボタンに変わり、次のカット位置を指定できるようになります。「ここまでカット」を押すまで動画の最後までになっていたカット終了点が、「ここまでカット」で終了点を指定すると更新されるイメージです。文章だと難しいと思いますが、触ってみると分かると思います。
「カットする箇所」はリストボックスになっています。ここで、どこからどこまでがカット対象として選択されているかを確認できます。リストボックスでカット位置を選んでいる状態でデリートキーを押すか、右クリックして「削除」を実行すると、選択したカットを取り消すことができます。
カット編集画面では、ショートカットキーを利用できます。
- ctrl+p: 再生/停止
- ctrl+f: 進める
- ctrl+b: 戻す
- ctrl+i: 間隔調整
- ctrl+g: 指定時間へ
- ctrl+j: ここからカット/ここまでカット

# 編集の仕組み
このソフトは、内部で FFmpeg というツールを使用しています。これは、コマンドを渡すといろんな動画の操作をやってくれるものです。

# 作業フォルダについて
このソフトの入っているフォルダの中に、「temp」という作業フォルダが自動的に作られます。この中には、動画編集の途中で必要になる一時的なファイルが入ります。
ソフトを終了するとき、作業フォルダをきれいにするかどうかを聞いてくることがあります。「はい」を押すと、 temp フォルダを中身ごと削除してから終了してくれます。普通は「はい」を推してきれいにして問題ありません。エラーが起きたときに、実行ログや中間結果を助け出したい場合は、「いいえ」を選んでフォルダを残してから、中身を手動で見ることができます。
作業フォルダの中には、いくつかのサブフォルダができます。
logs には、 FFmpeg の実行結果が入ります。 FFmpeg がエラーを出したときは、これを見ることで原因を特定できます。
concats には、動画をカット編集する際、途中のステップとして動画を細切れにしたときの作業結果が入ります。特にこのフォルダの中が大きくなりやすいと思います。
